# UI Framework Base

The foundation of the entire UI framework. All UI entry, exit, red dot system, etc., are controlled by this class.
:::danger[Important Notes to Avoid Errors During Import]
    Ensure that Dotween (free or paid version) is correctly installed, and verify that Addressable is properly installed.  
    If the red dot system is not required, it is recommended to comment out the red dot-related code.
:::

## Fields and Parameters
| Name | Type | Access Level | Description |
| :-: | :-: | :-: | :-: |
| startShowEvent | Func | private | Event triggered at the start of displaying the UI |
| completeShowEvent | Func | private | Event triggered upon completing the display of the UI |
| startHideEvent | Func | private | Event triggered at the start of hiding the UI |
| completeHideEvent | Func | private | Event triggered upon completing the hiding of the UI |
| startMoveEvent | Func | private | Event triggered at the start of moving the UI |
| completeMoveEvent | Func | private | Event triggered upon completing the movement of the UI |
| simpleTween | Tween | private | General Dotween animation used for movement |
| redPointhandle | AsyncOperationHandle | private | Dedicated red dot for a single UI Framework associated with this UI |

:::warning[Notes to Consider]
    The structure of `Func`: `Func<object, object>` 
    The first parameter of `Func` is the default parameter for `Action`. Generally, we use the second parameter, which is the UI itself (depending on the specific context).
:::

:::warning[Notes to Consider]
    The `ShowUI()` and `HideUI()` methods of the UI need to check whether the UI itself is activated. Therefore, they should be called through the parent or a master script for control. This design is intended to hierarchically manage the UI and avoid issues such as UI 1 directly calling UI 2.
:::

## Usage Examples
### Example of a UI Manager class  

```csharp title="UIFrameworkExampleManager.cs"
public class UIManager : MonoBehavior
{
    // Parameters and fields, etc...
    public SubUI subUI;

    private void Start() {
        subUI.SetUI(true);
    }

    private void OnApplicationQuit() {
        subUI.SetUI(false);
    }
}
```

### Example of a UI Framework class 

```csharp title="UIFrameworkExampleSubUI.cs"
public class SubUI : UIFrameworkBase
{
    // Parameters and fields, etc...

    // Override OnEnable and OnDisable, etc.

    public void SetUI(bool isShow)
    {
        if (isShow)
        {
            // Calculate the position to move to. Generally, switch to Debug mode in Hierarchy to check the anchored position. Here, (0, 0) is used as the endpoint.
            ShowUI(UiShowHideType.Slide, UiShowHideDirection.Left, 1f, UiShowHideBehavior.Synchronize);
        }
        else
        {
            // Calculate the position to move to. Generally, switch to Debug mode in Hierarchy to check the anchored position. Here, (0, 0) is used as the endpoint.
            HideUI(UiShowHideType.Slide, UiShowHideDirection.Left, 1f, UiShowHideBehavior.Synchronize);
        }
    }
}
```

### Example of Red Points
```csharp title="UIFrameWorkExampleRedPoint.cs"
private bool isRead = false;
    protected bool b_IsRead
    {
        get { return isRead; }
        set
        {
            isRead = value;
            //Inverted bool, cause there's no red point when read, and a red point when unread
            UseRedPoint(() => !isRead);
        }
    }
    public override void OnEnable()
    {
        base.OnEnable();
        b_IsRead = false;
        //btn respresents a UI button somewhere
        btn.onClick.AddListener(() =>
        {
            Camera.main.GetComponent<CameraPerspective>().MoveCameraTo(locator.position, 500f);
            b_IsRead = true;
        });
    }
```
